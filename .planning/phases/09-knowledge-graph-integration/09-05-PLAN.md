**graph_ingestor.py** - `GraphIngestor` class:

Constructor takes:
- `adapter: GraphAdapter` (the storage backend)
- `fact_store: FactStore` (to fetch full fact data)
- `verification_store: VerificationStore` (to fetch verification results)
- `classification_store: ClassificationStore` (to fetch classifications)
- `config: GraphConfig` (for relationship extraction settings)
- `bus: MessageBus | None = None` (optional for event-driven mode)

Uses structlog for logging, consistent with VerificationPipeline pattern.

Methods:

1. `register(bus: MessageBus | None = None) -> None`:
   Register as handler for `verification.complete` events on the MessageBus.
   Uses the existing `subscribe_to_pattern()` API from bus.py:
   ```python
   target_bus = bus or self._bus
   target_bus.subscribe_to_pattern(
       "graph_ingestor", "verification.complete",
       self._on_verification_complete
   )
   ```
   If no bus provided and self._bus is None, log warning and return (standalone mode).

2. `async _on_verification_complete(message: dict) -> None`:
   Event handler for verification.complete messages.
   - Extract `fact_id` and `investigation_id` from `message["payload"]`
   - Call `ingest_fact(investigation_id, fact_id)`
   - Log success/failure with structlog

3. `async ingest_fact(investigation_id: str, fact_id: str) -> dict[str, int]`:
   Ingest a single verified fact into the graph:
   - Fetch fact from FactStore: `await self._fact_store.get_fact(fact_id)` -- if FactStore uses sync API, handle accordingly (check actual FactStore interface)
   - Fetch verification result from VerificationStore: `await self._verification_store.get_result(investigation_id, fact_id)`
   - Fetch classification from ClassificationStore (if available)
   - Create FactMapper with investigation_id
   - Map fact to nodes/edges via `mapper.map_fact()`
   - Create RelationshipExtractor with config
   - Extract additional relationships (pass existing facts from FactStore for context)
   - Combine mapper edges + extractor edges
   - Call `adapter.batch_merge_nodes()` grouped by label
   - Call `adapter.batch_merge_relationships()` for all edges
   - Log stats and return {"nodes_merged": N, "edges_merged": M}

4. `async ingest_investigation(investigation_id: str) -> dict[str, int]`:
   Bulk ingest all verified facts for an investigation:
   - Fetch all verification results from VerificationStore: `await self._verification_store.get_all_results(investigation_id)`
   - Filter to CONFIRMED and SUPERSEDED status (skip REFUTED, UNVERIFIABLE, PENDING)
   - For each result, fetch the fact and classification
   - Use FactMapper.map_facts_batch() for batch entity resolution
   - Extract relationships for all facts
   - Batch merge all nodes and edges
   - Return total stats

5. `async ingest_investigation_all(investigation_id: str) -> dict[str, int]`:
   Like ingest_investigation but includes ALL facts regardless of verification status (for complete graph building).

**graph_pipeline.py** - `GraphPipeline` class:

Follows the same pattern as VerificationPipeline (lazy init, event registration, standalone mode):

Constructor takes optional pre-configured components:
- `graph_ingestor: GraphIngestor | None = None`
- `adapter: GraphAdapter | None = None`
- `fact_store: FactStore | None = None`
- `verification_store: VerificationStore | None = None`
- `classification_store: ClassificationStore | None = None`
- `config: GraphConfig | None = None`

Methods:

1. `_get_ingestor() -> GraphIngestor`:
   Lazy-init GraphIngestor with shared stores and adapter. If adapter is None, check config: if use_networkx_fallback, create NetworkXAdapter; otherwise create Neo4jAdapter.

2. `async on_verification_complete(investigation_id: str, verification_summary: dict[str, Any]) -> dict[str, Any]`:
   Handler for verification.complete events. Delegates to GraphIngestor.ingest_investigation(investigation_id).
   Returns ingestion stats.

3. `async run_ingestion(investigation_id: str, include_all: bool = False) -> dict[str, Any]`:
   Standalone mode. If include_all, uses ingest_investigation_all; otherwise ingest_investigation.

4. `def register_with_pipeline(investigation_pipeline: Any) -> None`:
   Register as handler for verification.complete events, same pattern as VerificationPipeline.
   ```python
   investigation_pipeline.on_event(
       "verification.complete",
       self.on_verification_complete
   )
   ```

5. `async query(query_type: str, **kwargs) -> QueryResult`:
   Convenience method for querying the graph. Delegates to adapter query methods based on query_type string.
