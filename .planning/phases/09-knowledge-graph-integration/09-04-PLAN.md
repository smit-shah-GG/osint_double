Review and harden the four query method implementations in both adapters. Fix any issues found during test development.

**NetworkXAdapter refinements:**

1. `query_entity_network()`:
   - Build node key from entity_id: `"Entity:{entity_id}"`
   - If node doesn't exist, return empty QueryResult (not raise)
   - Use BFS with depth limit `max_hops` to collect all reachable nodes (both directions -- the graph is directed but entity networks need bidirectional traversal)
   - Filter by investigation_id if provided: only include nodes where `properties.get("investigation_id") == investigation_id`
   - Collect all edges between returned nodes
   - Build QueryResult with query_type="entity_network", metadata includes entity_id, max_hops, investigation_id

2. `query_corroboration_clusters()`:
   - Find all edges with rel_type in (CORROBORATES, CONTRADICTS) where at least one endpoint has matching investigation_id
   - Group connected facts into clusters (use networkx connected_components on the subgraph of these edges)
   - Each cluster becomes a set of nodes + their connecting edges
   - Build QueryResult with query_type="corroboration_clusters", metadata includes cluster_count, investigation_id

3. `query_timeline()`:
   - Find entity node: `"Entity:{entity_id}"`
   - Find all Fact nodes connected via MENTIONS edges (reverse direction: Fact -MENTIONS-> Entity, so traverse incoming edges of Entity)
   - Filter by investigation_id if provided
   - Filter facts with non-None `temporal_value` property
   - Sort by temporal_value ascending (string sort works for ISO-format dates)
   - Build QueryResult with query_type="timeline", metadata includes entity_id, fact_count

4. `query_shortest_path()`:
   - Build node keys: `"Entity:{from_entity_id}"` and `"Entity:{to_entity_id}"`
   - If either node doesn't exist, return empty QueryResult
   - Use `nx.shortest_path()` on undirected view of graph (shortest path ignores direction)
   - If no path exists, return empty QueryResult (not raise)
   - Collect all nodes and edges along the path
   - Build QueryResult with query_type="shortest_path", metadata includes from_id, to_id, path_length

**Neo4jAdapter refinements:**
- Ensure Cypher queries handle `$investigation_id` being None by using `WHERE $investigation_id IS NULL OR node.investigation_id = $investigation_id` pattern
- Ensure result conversion: Neo4j Node objects -> GraphNode, Neo4j Relationship objects -> GraphEdge
- Add a private `_to_graph_node(record)` and `_to_graph_edge(record)` helper for consistent conversion
- Handle empty result sets gracefully (return empty QueryResult)
