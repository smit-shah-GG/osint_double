7. Query templates for 4 essential patterns:

**neo4j_adapter.py** - `Neo4jAdapter` class:

Per RESEARCH.md patterns:
- Constructor takes `GraphConfig` (not raw strings). Creates `AsyncGraphDatabase.driver()` with auth from config. Stores `_database` from config.
- `initialize()`: verify_connectivity(), then run all SCHEMA_INIT_QUERIES in a session
- `close()`: driver.close()
- `merge_node()`: Single MERGE using `execute_query()` with `database_` param (ALWAYS pass database_ per RESEARCH.md pitfall 3)
- `merge_relationship()`: MERGE nodes separately, then MERGE relationship (per RESEARCH.md pitfall 1 -- NEVER MERGE on full patterns)
- `batch_merge_nodes()`: UNWIND pattern from RESEARCH.md. Chunk into config.batch_size batches. The label is injected via safe f-string (validated against allowlist: Fact, Entity, Source, Investigation, Classification). All property values use $params.
- `batch_merge_relationships()`: UNWIND with separate MERGE for from/to nodes, then relationship
- `delete_node()`: DETACH DELETE
- Query methods: Execute the corresponding cypher_queries constant, convert results to QueryResult using GraphNode/GraphEdge constructors. Handle empty results gracefully.
- `execute_cypher()`: Raw query passthrough with parameters.

Critical implementation rules:
- NEVER share AsyncSession across tasks (per RESEARCH.md pitfall 2)
- ALWAYS parameterize values (per RESEARCH.md pitfall 5)
- ALWAYS bound variable-length paths (per RESEARCH.md pitfall 4)
- Use structlog for logging, consistent with other adapters

Implement `async with` context manager support (__aenter__/__aexit__) that calls initialize/close.

**networkx_adapter.py** - `NetworkXAdapter` class:

Implements GraphAdapter Protocol using `networkx.MultiDiGraph`:

- Constructor: Creates empty MultiDiGraph. Initializes `_node_index: dict[str, dict]` for key-property uniqueness enforcement (per RESEARCH.md pitfall 6 -- NetworkX MUST manually enforce uniqueness).
- `initialize()`: No-op (no constraints to create), but sets `_initialized = True`.
- `close()`: Clears graph and index.
- `merge_node()`: Check `_node_index` for existing node with same `{label}:{properties[key_property]}`. If exists: update properties (ON MATCH behavior). If not: add node (ON CREATE behavior). Return node key.
- `merge_relationship()`: Check if edge with same rel_type already exists between from/to. If yes: update properties. If no: add edge. Per RESEARCH.md pattern.
- `batch_merge_nodes()`: Loop calling merge_node (no perf benefit in NetworkX, but maintains interface parity). Return count.
- `batch_merge_relationships()`: Loop calling merge_relationship. Return count.
- `delete_node()`: Remove node and all edges. Return True/False.
- `query_entity_network()`: Use networkx ego_graph or BFS to find nodes within max_hops. Filter by investigation_id if provided. Convert to QueryResult.
- `query_corroboration_clusters()`: Iterate edges with type CORROBORATES or CONTRADICTS, filter by investigation_id, group into connected components. Convert to QueryResult.
- `query_timeline()`: Find all Fact nodes connected to entity via MENTIONS edges, filter those with temporal_value, sort by temporal_value ascending. Convert to QueryResult.
- `query_shortest_path()`: Use `nx.shortest_path()` between two entity node keys. Convert path to QueryResult with nodes and edges along path.
- `execute_cypher()`: Raise `NotImplementedError("Raw Cypher not supported by NetworkX adapter. Use high-level query methods.")` per RESEARCH.md.

Node key format: `"{label}:{properties[key_property]}"` -- consistent between both adapters.

**Update __init__.py** to export Neo4jAdapter, NetworkXAdapter.

**test_networkx_adapter.py** - Comprehensive tests using pytest + pytest-asyncio:

Test categories:
1. **Merge semantics**: merge_node creates on first call, updates on second call with same key. Verify properties updated. Verify count.
2. **Batch operations**: batch_merge_nodes with 10 nodes returns 10. Duplicates are merged not duplicated.
3. **Relationship merge**: merge_relationship creates edge. Second call with same nodes+type updates properties.
4. **No full-pattern MERGE pitfall**: Verify that merging a relationship between existing nodes does NOT create duplicate nodes.
5. **Query entity network**: Create entity with connected facts, verify N-hop traversal returns correct nodes.
6. **Query corroboration clusters**: Create two facts with CORROBORATES edge, verify cluster returned.
7. **Query timeline**: Create entity with temporally-marked facts, verify chronological order.
8. **Query shortest path**: Create A-B-C chain, verify shortest path from A to C.
9. **Delete node**: Verify node and edges removed.
10. **execute_cypher raises**: Verify NotImplementedError.
11. **Cross-investigation filtering**: Verify investigation_id filter works on queries.

Use `@pytest.fixture` for adapter setup. All tests are `async def` with `@pytest.mark.asyncio`.
