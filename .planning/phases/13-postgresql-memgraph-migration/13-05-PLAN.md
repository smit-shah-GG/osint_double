  <name>Task 1: Migrate ClassificationStore + VerificationStore</name>
  <files>
    osint_system/data_management/classification_store.py
    osint_system/data_management/verification_store.py
  </files>
  <action>
    **ClassificationStore migration** (698 lines):

    Constructor changes:
    - Remove `persistence_path` parameter, add `session_factory: async_sessionmaker[AsyncSession]`
    - Remove `self._storage`, `self._lock`, `self._persistence_path`
    - Remove `_save_to_file`, `_load_from_file`

    Index replacement -- the existing store has in-memory indexes:
    - `flag_index` (dubious flag type -> list[fact_id]) becomes: `SELECT WHERE dubious_flags @> '["phantom"]'::jsonb` or SQL query on tier/dubious_flags JSONB
    - `tier_index` (tier -> list[fact_id]) becomes: `SELECT WHERE tier = $tier`

    Key methods to migrate (read existing file for ALL methods):
    - `save_classification(investigation_id, classification)` -> INSERT/UPDATE via ClassificationModel. Classification is a FactClassification Pydantic model or dict.
    - `save_classifications(investigation_id, classifications)` -> bulk INSERT
    - `get_classification(investigation_id, fact_id)` -> SELECT WHERE (investigation_id, fact_id)
    - `get_all_classifications(investigation_id)` -> SELECT WHERE investigation_id, return dict keyed by fact_id
    - `get_by_tier(investigation_id, tier)` -> SELECT WHERE tier
    - `get_by_flag(investigation_id, flag)` -> SELECT WHERE dubious_flags contains flag
    - `get_priority_queue(investigation_id)` -> SELECT ORDER BY priority_score DESC
    - `reclassify(investigation_id, fact_id, new_classification)` -> UPDATE
    - `get_statistics(investigation_id)` -> aggregate counts by tier
    - `delete_investigation(investigation_id)` -> DELETE WHERE investigation_id
    - Any other public methods

    **VerificationStore migration** (246 lines -- smallest store):

    Constructor changes:
    - Remove `persistence_path`, add `session_factory: async_sessionmaker[AsyncSession]`
    - Remove `self._results`, `self._lock`, `self._persistence_path`
    - Remove `_load_from_file` (already identified as broken in prior sessions -- Postgres fixes this structurally)

    Key methods to migrate:
    - `save_result(result: VerificationResult)` -> INSERT/UPSERT via VerificationModel. Note: VerificationResult is a Pydantic model with investigation_id, fact_id, status, evidence, etc.
    - `get_result(investigation_id, fact_id)` -> SELECT WHERE (investigation_id, fact_id), return VerificationResultRecord or None
    - `get_results(investigation_id)` -> SELECT WHERE investigation_id, return dict keyed by fact_id
    - `get_by_status(investigation_id, status)` -> SELECT WHERE status
    - `get_statistics(investigation_id)` -> aggregate counts by status
    - `delete_investigation(investigation_id)` -> DELETE WHERE investigation_id
    - Any other public methods

    **For both stores:**
    - Preserve exact return types (dicts, Pydantic models, whatever the original returns)
    - Use `async with self._session_factory() as session:` pattern per method
    - Remove all JSON file I/O
    - Remove asyncio.Lock (database handles concurrency)
  </action>

  <files>osint_system/reporting/report_store.py</files>
  <action>
    Rewrite ReportStore (446 lines) to use SQLAlchemy async sessions.

    **Constructor changes:**
    - Keep `output_dir` parameter (still needed for writing .md and .pdf files to disk)
    - Remove `persistence_path` parameter
    - Add `session_factory: async_sessionmaker[AsyncSession]` parameter
    - Add `embedding_service: EmbeddingService | None = None` parameter. Import EmbeddingService from `osint_system.data_management.embeddings`. Store as `self._embedding_service`.
    - Remove `self._reports` (in-memory dict), `self._lock`, `self._persistence_path`
    - Remove `_save_metadata`, `_load_metadata` methods (JSON metadata persistence)

    **Key methods to migrate** (read existing file for ALL methods):
    - `save_report(investigation_id, markdown_content, synthesis=None)` -> INSERT via ReportModel. Content hash dedup: compute SHA256 of markdown_content, check if latest version has same hash, skip if identical. Auto-increment version number per investigation. Still write .md file to disk (output_dir) for PDF generation compatibility. **CRITICAL: If self._embedding_service is not None, extract the executive summary from synthesis (synthesis.executive_summary if synthesis is a Pydantic model, or synthesis.get("executive_summary") if dict) and compute embedding via `self._embedding_service.embed(executive_summary)`. Set the embedding on the ReportModel instance's embedding column (Vector(1024)) before committing. If executive_summary is empty/None or embedding_service is None, leave embedding column as NULL.** Return ReportRecord.
    - `get_latest(investigation_id)` -> SELECT WHERE investigation_id ORDER BY version DESC LIMIT 1, return ReportRecord or None
    - `get_version(investigation_id, version)` -> SELECT WHERE (investigation_id, version)
    - `list_versions(investigation_id)` -> SELECT WHERE investigation_id ORDER BY version, return list of ReportRecord
    - `has_changed(investigation_id, new_content)` -> compare SHA256 of new_content against latest version's content_hash
    - `get_report_count()` -> SELECT COUNT(DISTINCT investigation_id)
    - Any other public methods

    **ReportRecord preservation:** The existing ReportRecord is a Pydantic model. The PostgreSQL-backed store must return ReportRecord instances (or dicts that match), not raw ORM objects. ReportModel.to_dict() should produce data compatible with ReportRecord construction.

    **File output preserved:** The store still writes .md and .pdf files to output_dir for backward compatibility with the PDF renderer and dashboard. The database is the source of truth for report content and versioning; files are artifacts.

    **Remove JSON metadata persistence:** Delete `_save_metadata`, `_load_metadata`. Version tracking is now in PostgreSQL via the (investigation_id, version) unique constraint.
