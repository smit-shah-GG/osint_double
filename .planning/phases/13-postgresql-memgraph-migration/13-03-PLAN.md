  <action>
    1. Create `memgraph_queries.py` by adapting `cypher_queries.py` with these Memgraph syntax changes:

       **Schema init queries** -- rewrite all SCHEMA_INIT_QUERIES:
       - Constraints: `CREATE CONSTRAINT ON (f:Fact) ASSERT f.fact_id IS UNIQUE;` (not Neo4j's FOR/REQUIRE syntax). NO `IF NOT EXISTS` -- Memgraph does not support it. Caller must wrap in try/except.
       - Indexes: `CREATE INDEX ON :Fact(fact_id);` (not Neo4j's named index syntax). Create SEPARATE indexes even for constrained properties (Memgraph does NOT auto-create backing indexes for constraints -- Pitfall 4 from RESEARCH.md).
       - No TEXT INDEX (Memgraph doesn't support it) -- use label-property index on :Entity(name) instead.
       - No relationship property indexes (Memgraph doesn't support it) -- drop the CORROBORATES weight index entirely.
       - Full list of constraints: fact_id, entity_id, source_id, investigation_id (4 constraints)
       - Full list of indexes: all 4 constrained props + investigation_id on Fact, entity_type on Entity, canonical on Entity, name on Entity, temporal_value on Fact (total ~13 indexes)

       **Node MERGE** -- replace `datetime()` with `localDateTime()`:
       - MERGE_NODE: same pattern but `n.created_at = localDateTime()` and `n.updated_at = localDateTime()`
       - BATCH_MERGE_NODES: same datetime replacement
       - MERGE_RELATIONSHIP: same datetime replacement
       - BATCH_MERGE_RELATIONSHIPS: same datetime replacement
       - DELETE_NODE: no changes needed

       **Query patterns**:
       - QUERY_ENTITY_NETWORK: no changes needed (same syntax)
       - QUERY_CORROBORATION_CLUSTERS: no changes needed
       - QUERY_TIMELINE: no changes needed
       - QUERY_SHORTEST_PATH: REWRITE -- change `shortestPath((a)-[*..10]-(b))` to Memgraph BFS syntax: `(a:Entity {{entity_id: $from_id}})-[*BFS ..10]-(b:Entity {{entity_id: $to_id}})`. See RESEARCH.md Pattern 4 and Pitfall 9.

    2. Create `mage_algorithms.py`:
       - Define Cypher query constants for 3 MAGE algorithms:
         - `PAGERANK_QUERY`: `CALL pagerank.get() YIELD node, rank SET node.rank = rank RETURN count(node) AS nodes_ranked`
         - `COMMUNITY_DETECTION_QUERY`: `CALL community_detection.get() YIELD node, community_id SET node.community = community_id RETURN count(node) AS nodes_assigned`
         - `BETWEENNESS_QUERY`: `CALL betweenness_centrality.get(TRUE, TRUE) YIELD node, betweenness_centrality SET node.betweenness = betweenness_centrality RETURN count(node) AS nodes_scored`
       - Async function `run_mage_analysis(adapter, investigation_id: str | None = None) -> dict[str, int]`:
         - Takes any GraphAdapter (will use execute_cypher)
         - Runs all 3 algorithms sequentially (not parallel -- they modify graph state)
         - Returns dict: {"pagerank": N, "communities": N, "betweenness": N}
         - Wraps each call in try/except: if MAGE module not available, log warning and skip (graceful degradation)
         - Use full-graph versions initially. Investigation-scoped subgraph projection is deferred per RESEARCH.md open question 3.
  </action>
