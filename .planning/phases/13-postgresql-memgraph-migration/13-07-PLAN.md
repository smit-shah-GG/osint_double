<task type="auto">
  <name>Task 2: JSON-to-PostgreSQL data migration script</name>
  <files>scripts/migrate_json_to_postgres.py</files>
  <action>
    Create `scripts/migrate_json_to_postgres.py` -- a one-time migration script that reads existing JSON-persisted investigation data and inserts into PostgreSQL.

    The script should:
    1. Scan `data/` directory for investigation directories (pattern: `data/inv-*/` or `data/*-*/`)
    2. For each investigation directory, look for JSON files: articles.json, facts.json, classifications.json, verifications.json, reports/ directory
    3. Read each JSON file and parse the data structures (they match the old store formats)
    4. Initialize database with `init_db()`
    5. For each investigation:
       - Insert articles via ArticleModel.from_dict()
       - Insert facts via FactModel.from_dict()
       - Insert classifications via ClassificationModel.from_dict()
       - Insert verifications via VerificationModel.from_dict()
       - Insert reports via ReportModel.from_dict()
    6. Use bulk inserts within transactions for performance
    7. Handle duplicates gracefully (skip already-migrated data)
    8. Log progress: "Migrating investigation {id}: {n} articles, {m} facts, ..."
    9. Print summary at end: total investigations, total records migrated

    The script should be runnable as:
    ```bash
    uv run python scripts/migrate_json_to_postgres.py
    ```

    Make it idempotent: running twice should not create duplicate records (use content_hash or fact_id checks).

    IMPORTANT: Read the existing JSON file formats by examining the _save_to_file methods in the ORIGINAL store files (before migration). The JSON format matches the in-memory dict structure.
  </action>
  <verify>
    - `uv run python -c "from scripts.migrate_json_to_postgres import *; print('migration script imports ok')"` or `uv run python scripts/migrate_json_to_postgres.py --dry-run` succeeds (if --dry-run flag added)
    - Script file exists at `scripts/migrate_json_to_postgres.py`
    - Script handles missing data/ directory gracefully (prints "No data directory found")
  </verify>
  <done>
    Migration script reads existing JSON investigation data and inserts into PostgreSQL. Idempotent, handles missing files, logs progress.
  </done>
