<task type="auto">
  <name>Task 1: Docker Compose + init.sql + Database Config</name>
  <files>
    docker-compose.yml
    init.sql
    osint_system/config/database_config.py
  </files>
  <action>
    1. Create `docker-compose.yml` at project root with two services:
       - `postgres`: image `pgvector/pgvector:pg17`, env vars POSTGRES_USER=osint, POSTGRES_PASSWORD=osint_dev_password, POSTGRES_DB=osint, port 5432:5432, volume `pgdata:/var/lib/postgresql/data`, mount `./init.sql:/docker-entrypoint-initdb.d/init.sql`, healthcheck using `pg_isready -U osint` (interval 5s, timeout 5s, retries 5)
       - `memgraph`: image `memgraph/memgraph-mage`, ports 7687:7687 and 7444:7444, volume `mgdata:/var/lib/memgraph`, command `["--bolt-server-name-for-init=Neo4j/5.2.0"]` for neo4j driver compat
       - Named volumes: pgdata, mgdata

    2. Create `init.sql` at project root:
       ```sql
       CREATE EXTENSION IF NOT EXISTS vector;
       ```

    3. Create `osint_system/config/database_config.py` using Pydantic BaseModel (NOT BaseSettings -- follow the same pattern as GraphConfig which uses from_env classmethod):
       - Fields: postgres_host (default "localhost"), postgres_port (default 5432), postgres_user (default "osint"), postgres_password (default "osint_dev_password"), postgres_db (default "osint"), pool_size (default 10, ge=1, le=50), max_overflow (default 20, ge=0, le=100)
       - Property `database_url` that returns `postgresql+asyncpg://{user}:{password}@{host}:{port}/{db}`
       - Property `sync_database_url` that returns `postgresql+psycopg://{user}:{password}@{host}:{port}/{db}` (needed for Alembic)
       - Classmethod `from_env()` loading from env vars: POSTGRES_HOST, POSTGRES_PORT, POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB, POSTGRES_POOL_SIZE, POSTGRES_MAX_OVERFLOW. Follow exact same pattern as GraphConfig.from_env() with .env file loading via python-dotenv.

    4. Install new dependencies:
       ```bash
       uv pip install "sqlalchemy[asyncio]>=2.0" asyncpg "psycopg[binary]>=3.1" pgvector alembic
       ```
       Update requirements.txt with `uv pip freeze > requirements.txt`

    5. Start Docker containers and verify health:
       ```bash
       docker compose up -d postgres memgraph
       ```
       Wait for PostgreSQL healthcheck to pass (poll `docker compose ps` or `docker compose exec postgres pg_isready -U osint` until success, max 30 seconds). Verify pgvector extension: `docker compose exec postgres psql -U osint -c "SELECT * FROM pg_extension WHERE extname = 'vector'"` must return a row.
  </action>

    - `uv run python -c "import sqlalchemy; import asyncpg; import pgvector; import alembic; print('all imports ok')"` succeeds

<task type="auto">
  <name>Task 2: Async SQLAlchemy Engine + Alembic Init</name>
  <files>
    osint_system/data_management/database.py

    migrations/script.py.mako

  <action>
    1. Create `osint_system/data_management/database.py`:
       - Import AsyncEngine, AsyncSession, async_sessionmaker, create_async_engine from sqlalchemy.ext.asyncio
       - Import DatabaseConfig from osint_system.config.database_config
       - Function `create_engine(config: DatabaseConfig) -> AsyncEngine`: creates async engine with config.database_url, pool_size=config.pool_size, max_overflow=config.max_overflow, pool_pre_ping=True, echo=False
       - Function `create_session_factory(engine: AsyncEngine) -> async_sessionmaker[AsyncSession]`: creates factory with expire_on_commit=False (CRITICAL for async -- see RESEARCH.md anti-patterns)
       - Module-level convenience: `_engine: AsyncEngine | None = None` and `_session_factory: async_sessionmaker[AsyncSession] | None = None`
       - Function `init_db(config: DatabaseConfig | None = None) -> async_sessionmaker[AsyncSession]`: initializes module-level engine and session factory from config (defaults to DatabaseConfig.from_env()). Returns the session factory. Idempotent -- returns existing factory if already initialized.
       - Function `get_session_factory() -> async_sessionmaker[AsyncSession]`: returns module-level factory, raises RuntimeError if not initialized
       - Function `get_engine() -> AsyncEngine`: returns module-level engine, raises RuntimeError if not initialized
       - Async function `close_db() -> None`: disposes engine, resets module-level vars to None

    2. Initialize Alembic:
       ```bash
       cd /home/kondraki/personal/osint_double && uv run alembic init -t async migrations
       ```
       This creates alembic.ini, migrations/env.py, migrations/script.py.mako, migrations/versions/

    3. Edit `alembic.ini`:
       - Set `sqlalchemy.url = postgresql+asyncpg://osint:osint_dev_password@localhost:5432/osint`
       - Set `script_location = migrations`

    4. Edit `migrations/env.py`:
       - Import `from osint_system.data_management.models.base import Base` (will be created in Plan 02)
       - Set `target_metadata = Base.metadata`
       - The async template already handles `run_async_migrations()` -- just wire in the metadata import
       - Add a comment noting that models/__init__.py must import all model classes for autogenerate to detect them

    Do NOT create ORM models yet (that's Plan 02). Docker containers are already running from Task 1.
  </action>

    - `test -f alembic.ini && test -f migrations/env.py && echo 'alembic initialized'` prints 'alembic initialized'
    - `grep 'asyncpg' alembic.ini` finds the connection URL
  </verify>
