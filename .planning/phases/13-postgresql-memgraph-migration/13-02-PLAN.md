    Create the `osint_system/data_management/models/` package.

    1. **base.py** -- DeclarativeBase + common timestamp mixin:
       - `from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column`
       - `class Base(DeclarativeBase): pass`
       - `class TimestampMixin`: two mapped_columns: `created_at: Mapped[datetime]` (server_default=func.now()), `updated_at: Mapped[datetime | None]` (onupdate=func.now(), nullable=True)

    2. **article.py** -- ArticleModel:
       - Tablename: `articles`
       - Columns: id (int PK), article_id (String(64), unique, indexed), investigation_id (String(64), indexed), url (Text), title (Text, nullable), content (Text, nullable), published_date (String(64), nullable), source_name (String(256), nullable), source_domain (String(256), nullable)
       - JSONB columns: source_metadata (JSONB, nullable), article_metadata (JSONB, nullable)
       - pgvector: embedding (Vector(1024), nullable)
       - tsvector: content_tsvector = Computed("to_tsvector('english', COALESCE(title, '') || ' ' || COALESCE(content, ''))", persisted=True) + GIN index
       - Uses TimestampMixin
       - `from_dict(cls, data: dict, investigation_id: str) -> ArticleModel`: maps from existing ArticleStore dict format. The existing format stores url, title, content, published_date, source (dict with name, domain), metadata (dict), stored_at. Generate article_id as hash of url if not present.
       - `to_dict(self) -> dict`: returns exact same dict shape as ArticleStore currently returns (url, title, content, published_date, source, metadata, stored_at)

    3. **fact.py** -- FactModel:
       - Tablename: `facts`
       - Columns: id (int PK), fact_id (String(64), unique, indexed), investigation_id (String(64), indexed), content_hash (String(64), indexed), claim_text (Text, NOT NULL), assertion_type (String(32), default="statement"), source_url (Text, nullable), extraction_confidence (Float, nullable), claim_clarity (Float, nullable)
       - JSONB columns: entities (JSONB, default=[]), provenance (JSONB, nullable), quality_metrics (JSONB, nullable), temporal (JSONB, nullable), numeric (JSONB, nullable), relationships (JSONB, default=[]), variants (JSONB, default=[]), claim_data (JSONB, nullable -- stores entire claim sub-object for fields not in columns)
       - pgvector: embedding (Vector(1024), nullable)
       - tsvector: claim_tsvector = Computed("to_tsvector('english', COALESCE(claim_text, ''))", persisted=True) + GIN index
       - HNSW index on embedding: postgresql_using="hnsw", postgresql_with={"m": 16, "ef_construction": 64}, postgresql_ops={"embedding": "vector_cosine_ops"}
       - Uses TimestampMixin
       - `from_dict(cls, data: dict, investigation_id: str) -> FactModel`: maps from FactStore dict format. Read the existing FactStore data structure carefully -- fact_id, content_hash, claim (nested dict with text, type, etc.), variants, stored_at. Extract claim.text -> claim_text, claim.type -> assertion_type.
       - `to_dict(self) -> dict`: returns exact same dict shape as existing store

    4. **entity.py** -- EntityModel (for pgvector embedding on entity canonical names):
       - Tablename: `entities`
       - Columns: id (int PK), entity_id (String(64), unique, indexed), investigation_id (String(64), indexed), name (Text), entity_type (String(64), nullable), canonical (Text, nullable)
       - JSONB: entity_metadata (JSONB, nullable)
       - pgvector: embedding (Vector(1024), nullable)
       - Uses TimestampMixin
       - `from_dict(cls, data: dict, investigation_id: str) -> EntityModel`
       - `to_dict(self) -> dict`

    5. **__init__.py** -- import ALL model classes (CRITICAL for Alembic autogenerate):
       ```python

           "Base", "ArticleModel", "FactModel", "ClassificationModel",

       ]
       ```
       NOTE: ClassificationModel, VerificationModel, ReportModel don't exist yet (Task 2). Use a try/except ImportError for those in __init__.py for now, OR create all models in a single pass. Recommend creating stubs for classification/verification/report that Task 2 will flesh out.

    CRITICAL: Read each existing store file carefully to understand the EXACT dict shapes returned. The to_dict() methods MUST produce identical output. Mismatched keys will break every pipeline and API consumer.
