       PyTorch with CUDA should already be available or will be pulled as a dependency. Update requirements.txt.

    2. Create `osint_system/data_management/embeddings.py`:

       ```python
       class EmbeddingService:
       ```

       - **Constructor** `__init__(self, model_name: str = "Alibaba-NLP/gte-large-en-v1.5", device: str | None = None)`:
         - Load model via `SentenceTransformer(model_name, trust_remote_code=True)`
         - If device is None: auto-detect CUDA (`"cuda" if torch.cuda.is_available() else "cpu"`)
         - Move model to device
         - Store model dimension (should be 1024) for validation
         - Log model name, device, and dimension at INFO level

       - **`async def embed(self, text: str) -> list[float]`**:
         - Run `self._model.encode(text, normalize_embeddings=True)` in a thread executor (CPU-bound work must not block event loop)
         - Return as Python list of floats (pgvector expects this)
         - Handle empty string: return zero vector of correct dimension

       - **`async def embed_batch(self, texts: list[str], batch_size: int = 32) -> list[list[float]]`**:
         - Run `self._model.encode(texts, normalize_embeddings=True, batch_size=batch_size)` in executor
         - Return as list of lists
         - Handle empty list: return empty list

       - **`@property def dimension(self) -> int`**: returns model dimension (1024)

       - **`def embed_sync(self, text: str) -> list[float]`**: synchronous version for use in migration scripts or non-async contexts

       Use `asyncio.get_running_loop().run_in_executor(None, ...)` for async wrapping. The model.encode() call is CPU/GPU-bound and should not run on the event loop.

    3. Create `tests/data_management/test_embeddings.py`:
       - Test that EmbeddingService can be constructed (use CPU device in tests to avoid CUDA requirement in CI)
       - Test that embed() returns a list of 1024 floats
       - Test that embed_batch() returns correct number of vectors
       - Test that empty string produces a vector (not an error)
       - Test that embed_sync() works
       - Mock SentenceTransformer in tests to avoid downloading the 1.2GB model during test runs. Mock `encode()` to return numpy arrays of the correct shape.

    IMPORTANT: The model download (1.2GB) will happen on first construction. This is expected. The test file MUST mock the model to avoid this download in CI/test runs.
