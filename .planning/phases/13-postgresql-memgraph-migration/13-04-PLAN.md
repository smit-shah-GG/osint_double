<objective>
Migrate ArticleStore and FactStore from in-memory+JSON to SQLAlchemy+PostgreSQL backends while preserving identical method signatures and return types. Wire EmbeddingService into both stores for pgvector column population. Add entity extraction to FactStore to populate the entities table.

Purpose: These are the two largest and most complex stores (426 and 693 lines respectively). They handle article persistence and fact storage with deduplication, variant linking, and indexed lookups. The migration swaps the dict+lock backend for SQLAlchemy async sessions while keeping every public method signature and return type identical. EmbeddingService (available since wave 2) populates pgvector columns. Entity extraction ensures the EntityModel table is not orphaned.

Output: Rewritten article_store.py and fact_store.py with PostgreSQL backends, embedding support, and entity extraction
</objective>

<task type="auto">
  <name>Task 1: Migrate ArticleStore to PostgreSQL with embedding support</name>
  <files>osint_system/data_management/article_store.py</files>
  <action>
    Rewrite ArticleStore to use SQLAlchemy async sessions instead of in-memory dicts.

    **Constructor changes:**
    - Remove `persistence_path` parameter
    - Add `session_factory: async_sessionmaker[AsyncSession]` parameter
    - Add `embedding_service: EmbeddingService | None = None` parameter (optional -- graceful if not provided)
    - Store as `self._session_factory` and `self._embedding_service`
    - Remove `self._storage`, `self._url_index`, `self._lock`, `self._persistence_path`
    - Remove `_save_to_file`, `_load_from_file` methods entirely

    **Embedding wiring in save methods:**
    - In `save_articles()`, after creating each ArticleModel via `from_dict()`, if `self._embedding_service` is not None, generate embedding from article content (title + content concatenation) and set `model.embedding = await self._embedding_service.embed(text)`. This populates the pgvector column for semantic search.
    - If embedding_service is None, leave embedding as NULL (graceful degradation).

    **Method migration pattern** (apply to EVERY public method):
    - Replace `async with self._lock:` with `async with self._session_factory() as session:`
    - Replace dict lookups with SQLAlchemy queries using `select()`, `where()`, etc.
    - Replace dict mutations with `session.add()`, `session.merge()`, `session.delete()`
    - Wrap mutations in `async with session.begin():` for auto-commit
    - Convert ORM results to dicts using `ArticleModel.to_dict()` before returning

    **Key methods to migrate** (read existing file to understand ALL methods):
    - `save_articles(investigation_id, articles, metadata)` -> INSERT via ArticleModel.from_dict(), skip duplicates by URL, embed content
    - `get_articles(investigation_id)` -> SELECT WHERE investigation_id, return list of dicts
    - `get_article_by_url(investigation_id, url)` -> SELECT WHERE url
    - `get_article_count(investigation_id)` -> SELECT COUNT
    - `get_all_investigations()` -> SELECT DISTINCT investigation_id
    - `delete_investigation(investigation_id)` -> DELETE WHERE investigation_id
    - Any other public methods that exist

    **Critical contract:** Every method must return the EXACT same types and dict shapes as the current implementation. Callers must not know the backend changed. Do NOT add new methods or change signatures (except the constructor which changes from persistence_path to session_factory + embedding_service).

    **URL dedup:** The existing store uses `_url_index` for O(1) URL dedup. Replace with a UNIQUE constraint or WHERE NOT EXISTS subquery on (investigation_id, url). ArticleModel should have a unique constraint on (investigation_id, url) or use INSERT ... ON CONFLICT DO NOTHING.

    **Remove JSON persistence:** Delete all `_save_to_file`, `_load_from_file`, and file I/O code. PostgreSQL is the single source of truth.
  </action>

<task type="auto">
  <name>Task 2: Migrate FactStore to PostgreSQL with embedding + entity extraction</name>
  <files>osint_system/data_management/fact_store.py</files>
  <action>
    Rewrite FactStore to use SQLAlchemy async sessions. This is the most complex store (693 lines) with multiple indexes and variant linking.

    **Constructor changes:**
    - Remove `persistence_path` parameter
    - Add `session_factory: async_sessionmaker[AsyncSession]` parameter
    - Add `embedding_service: EmbeddingService | None = None` parameter (optional -- graceful if not provided)
    - Store as `self._session_factory` and `self._embedding_service`
    - Remove `self._storage`, `self._fact_index`, `self._hash_index`, `self._source_index`, `self._lock`, `self._persistence_path`
    - Remove `_save_to_file`, `_load_from_file` methods

    **Embedding wiring in save_facts():**
    - After creating each FactModel via `from_dict()`, if `self._embedding_service` is not None, generate embedding from claim_text: `model.embedding = await self._embedding_service.embed(model.claim_text)`. This is MANDATORY when embedding_service is provided -- not optional.
    - If embedding_service is None, leave embedding as NULL.

    **Entity extraction in save_facts() -- populates EntityModel table:**
    - Each fact's JSONB `entities` field contains a list of entity dicts (e.g., `[{"name": "Russia", "type": "GPE"}, {"name": "NATO", "type": "ORG"}]`).
    - After saving each FactModel, iterate over the `entities` JSONB array and for each entity:
      1. Generate entity_id as a deterministic hash of (investigation_id, canonical or name, entity_type) to enable deduplication.
      2. UPSERT an EntityModel row: `INSERT ... ON CONFLICT (entity_id) DO UPDATE SET updated_at = now()`. Use SQLAlchemy's `insert().on_conflict_do_update()` with the `entity_id` unique constraint.
      3. If `self._embedding_service` is not None, embed the entity name/canonical and set `entity_model.embedding`.
    - This ensures the `entities` table is populated as facts are ingested -- no separate population step needed.
    - Import EntityModel: `from osint_system.data_management.models.entity import EntityModel`
    - Wrap entity extraction in try/except so a failure to extract entities does not abort the fact save. Log warnings on entity extraction errors.

    **Index replacement:**
    - `_fact_index` (fact_id -> fact) becomes: `SELECT WHERE fact_id = $id`
    - `_hash_index` (content_hash -> list[fact_id]) becomes: `SELECT WHERE content_hash = $hash`
    - `_source_index` (source_id -> list[fact_id]) becomes: JSONB query on provenance or a join

    **Key methods to migrate** (read existing file for ALL methods):
    - `save_facts(investigation_id, facts, investigation_metadata)` -> bulk INSERT via FactModel.from_dict(). Embed claim_text. Extract entities. Return dict with saved/updated/skipped/total counts.
    - `get_facts(investigation_id)` -> SELECT WHERE investigation_id, return dict of {fact_id: fact_dict}
    - `get_fact(investigation_id, fact_id)` -> SELECT WHERE fact_id
    - `get_fact_by_id(fact_id)` -> SELECT WHERE fact_id (cross-investigation lookup)
    - `get_facts_by_hash(content_hash)` -> SELECT WHERE content_hash
    - `link_variants(investigation_id, primary_id, variant_id)` -> UPDATE variants JSONB array
    - `get_fact_count(investigation_id)` -> SELECT COUNT
    - `update_fact(investigation_id, fact_id, updates)` -> UPDATE specific columns/JSONB
    - `delete_investigation(investigation_id)` -> DELETE WHERE investigation_id
    - Any other public methods

    **Variant linking:** The existing store appends variant_id to a list in the primary fact's `variants` field. In PostgreSQL, this is a JSONB array append operation. Use `func.jsonb_insert` or read-modify-write within a transaction.

    **Critical contract:** Return types must match exactly. `get_facts()` returns `Dict[str, Dict]` keyed by fact_id. `save_facts()` returns the stats dict.

    **Remove JSON persistence:** Delete all file I/O code.
  </action>
