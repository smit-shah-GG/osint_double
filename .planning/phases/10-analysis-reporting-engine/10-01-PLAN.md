This plan establishes both: the Pydantic schemas that define what analysis produces, and the DataAggregator that collects raw data from all stores into a single typed snapshot.

Output: analysis/ package with schemas and DataAggregator, config/analysis_config.py, tests.

**schemas.py** - Pydantic models for analysis output. Per RESEARCH.md Pattern 2:

1. `ConfidenceAssessment(BaseModel)`:
   - `level: Literal["low", "moderate", "high"]` - IC-style confidence language
   - `numeric: float = Field(..., ge=0.0, le=1.0)` - Numeric score
   - `reasoning: str` - Why this confidence level
   - `source_count: int` - Number of independent sources supporting
   - `highest_authority: float` - Highest authority score among sources

2. `KeyJudgment(BaseModel)`:
   - `judgment: str` - The analytical judgment statement
   - `confidence: ConfidenceAssessment`
   - `supporting_fact_ids: list[str]` - Fact IDs backing this judgment
   - `reasoning: str` - Analytical reasoning chain

3. `AlternativeHypothesis(BaseModel)`:
   - `hypothesis: str` - The alternative interpretation
   - `likelihood: Literal["unlikely", "possible", "plausible"]`
   - `supporting_evidence: list[str]` - Evidence points favoring this hypothesis
   - `weaknesses: list[str]` - Why this hypothesis may be wrong

4. `ContradictionEntry(BaseModel)`:
   - `description: str` - What the contradiction is
   - `fact_ids: list[str]` - Conflicting fact IDs
   - `resolution_status: Literal["resolved", "unresolved", "partially_resolved"]`
   - `resolution_notes: str = ""` - How it was resolved (if applicable)

5. `TimelineEntry(BaseModel)`:
   - `timestamp: str` - ISO-ish date string
   - `event: str` - Event description
   - `fact_ids: list[str]` - Supporting fact IDs
   - `confidence: ConfidenceAssessment`

6. `SourceInventoryEntry(BaseModel)`:
   - `source_id: str`
   - `source_domain: str`
   - `source_type: str`
   - `authority_score: float`
   - `fact_count: int` - Facts from this source
   - `last_accessed: str` - ISO timestamp

7. `InvestigationSnapshot(BaseModel)`:
   Container for all investigation data, ready for synthesis:
   - `investigation_id: str`
   - `objective: str = ""` - Investigation objective/target
   - `facts: list[dict[str, Any]]` - Raw fact dicts from FactStore
   - `classifications: list[dict[str, Any]]` - Classification dicts
   - `verification_results: list[dict[str, Any]]` - Verification result dicts
   - `graph_summary: dict[str, Any] = Field(default_factory=dict)` - Graph stats and key entities
   - `fact_count: int = 0`
   - `confirmed_count: int = 0`
   - `refuted_count: int = 0`
   - `unverifiable_count: int = 0`
   - `dubious_count: int = 0`
   - `source_inventory: list[SourceInventoryEntry] = Field(default_factory=list)`
   - `timeline_entries: list[TimelineEntry] = Field(default_factory=list)`
   - `created_at: datetime`
   - Helper method `token_estimate() -> int`: Rough estimate of total tokens when serialized (len(json) / 4)

8. `AnalysisSynthesis(BaseModel)`:
   Complete analysis output:
   - `investigation_id: str`
   - `executive_summary: str` - 1-2 paragraph executive brief
   - `key_judgments: list[KeyJudgment]`
   - `alternative_hypotheses: list[AlternativeHypothesis]`
   - `contradictions: list[ContradictionEntry]`
   - `implications: list[str]` - Strategic implications
   - `forecasts: list[str]` - Forward-looking assessments
   - `overall_confidence: ConfidenceAssessment`
   - `source_assessment: str` - Overall assessment of source quality/diversity
   - `snapshot: InvestigationSnapshot` - The data this analysis was based on
   - `generated_at: datetime = Field(default_factory=...)`
   - `model_version: str = ""` - Which LLM was used
   - `version: int = 1` - Report version number

All models include `model_config` with examples consistent with existing schema patterns.

**data_aggregator.py** - `DataAggregator` class:

Constructor takes:
- `fact_store: FactStore`
- `classification_store: ClassificationStore`
- `verification_store: VerificationStore`
- `graph_pipeline: GraphPipeline | None = None` (optional, graph queries skipped if None)

Uses structlog for logging, consistent with pipeline patterns.

Methods:

1. `async aggregate(investigation_id: str) -> InvestigationSnapshot`:
   Main method. Collects data from all stores into a single InvestigationSnapshot:

   a. Fetch facts: `await self._fact_store.retrieve_by_investigation(investigation_id)` -> extract facts list and metadata
   b. Fetch classification stats: `await self._classification_store.get_stats(investigation_id)` -> get counts. Also fetch all classifications for the investigation to include in snapshot.
   c. Fetch verification results: `await self._verification_store.get_all_results(investigation_id)` -> list of VerificationResultRecord. Serialize to dicts via `.model_dump(mode="json")`. Count by status.
   d. If graph_pipeline provided, query corroboration clusters and build graph_summary dict:
      ```python
      corr = await self._graph_pipeline.query("corroboration_clusters", investigation_id=investigation_id)
      graph_summary = {"node_count": corr.node_count, "edge_count": corr.edge_count, "clusters": len(corr.metadata.get("clusters", []))}
      ```
      Wrap in try/except — graph queries may fail if graph not populated yet. Log warning, continue with empty graph_summary.
   e. Build source inventory from facts: group facts by source_id, extract domain/type/authority from provenance and verification data. Create SourceInventoryEntry for each unique source.
   f. Build timeline entries from facts with temporal markers: extract temporal values, create TimelineEntry for each. Sort chronologically.
   g. Compute counts: fact_count, confirmed_count (from verification results with CONFIRMED status), refuted_count, unverifiable_count, dubious_count (from classifications with dubious_flags).

   Return populated InvestigationSnapshot.

2. `_build_source_inventory(facts: list[dict], verifications: list[dict]) -> list[SourceInventoryEntry]`:
   Group facts by source_id from provenance. For each source, extract domain/type from fact metadata or verification evidence. Count facts per source.

3. `_build_timeline(facts: list[dict], classifications: list[dict]) -> list[TimelineEntry]`:
   Extract temporal markers from facts. For each temporal marker, create TimelineEntry with the fact's claim text, sorted by temporal value. Generate ConfidenceAssessment based on classification credibility score if available.

4. `_count_by_verification_status(verifications: list[dict]) -> dict[str, int]`:
   Count verification results by status string.
