**app.py** - FastAPI application factory:

```python
def create_app(
    fact_store: FactStore | None = None,
    classification_store: ClassificationStore | None = None,
    verification_store: VerificationStore | None = None,
    report_store: ReportStore | None = None,
    report_generator: ReportGenerator | None = None,
    analysis_pipeline: AnalysisPipeline | None = None,
    config: AnalysisConfig | None = None,
) -> FastAPI:
```

- Create FastAPI instance with title="OSINT Dashboard"
- Mount static files: `app.mount("/static", StaticFiles(directory=static_dir))`
- Set up Jinja2Templates pointing to templates directory
- Store references on `app.state` for route access:
  ```python
  app.state.fact_store = fact_store or FactStore()
  app.state.classification_store = classification_store or ClassificationStore()
  app.state.verification_store = verification_store or VerificationStore()
  app.state.report_store = report_store or ReportStore()
  app.state.report_generator = report_generator
  app.state.analysis_pipeline = analysis_pipeline
  app.state.templates = templates
  ```
- Include route modules as APIRouter:
  ```python
  app.include_router(investigations.router, prefix="", tags=["investigations"])
  app.include_router(facts.router, prefix="/facts", tags=["facts"])
  app.include_router(reports.router, prefix="/reports", tags=["reports"])
  app.include_router(monitoring.router, prefix="/monitoring", tags=["monitoring"])
  app.include_router(api.router, prefix="/api", tags=["api"])
  ```
- Add health check endpoint: `GET /health` returns `{"status": "ok"}`
- Return app

Also provide a CLI entry point function:
```python
def run_dashboard(host: str = "127.0.0.1", port: int = 8080, **kwargs):
    import uvicorn
    app = create_app(**kwargs)
    uvicorn.run(app, host=host, port=port)
```

**Route modules:**

**investigations.py** - `router = APIRouter()`:

1. `GET /` - Investigation list:
   - Fetch `fact_store.list_investigations()`
   - For each, also fetch classification stats and verification stats
   - Render `investigations/list.html` with investigation summaries

2. `GET /investigation/{investigation_id}` - Investigation detail:
   - Fetch facts via `fact_store.retrieve_by_investigation(investigation_id)`
   - Fetch classification stats via `classification_store.get_stats(investigation_id)`
   - Fetch verification stats via `verification_store.get_stats(investigation_id)`
   - Check if report exists via `report_store.get_latest(investigation_id)`
   - Render `investigations/detail.html`

**facts.py** - `router = APIRouter()`:

1. `GET /{investigation_id}` - Fact list with filtering:
   - Query params: `tier` (critical/less_critical), `status` (confirmed/refuted/dubious/all), `page` (pagination)
   - Fetch facts, classifications, verifications for the investigation
   - Apply filters
   - Render `facts/list.html` with paginated fact table

**reports.py** - `router = APIRouter()`:

1. `GET /{investigation_id}` - View latest report:
   - Fetch latest report from report_store
   - If no report exists, show "No report generated" with generate button
   - Render `reports/view.html` with Markdown content (render to HTML in template using mistune or pass pre-rendered HTML)

2. `POST /{investigation_id}/generate` - Trigger report generation:
   - Call `analysis_pipeline.run_analysis(investigation_id)` if analysis_pipeline is available on app.state. Note: AnalysisPipeline.run_analysis() now auto-generates and saves the report via its internal report_generator and report_store if configured.
   - If analysis_pipeline is not available but report_generator is on app.state, fall back to generating a report from the latest stored synthesis (if available in report_store).
   - Redirect to report view (or return HTMX partial)

**monitoring.py** - `router = APIRouter()`:

1. `GET /status` - Pipeline monitoring dashboard:
   - Fetch global stats from FactStore and ClassificationStore via their `get_storage_stats()` methods (both have this method).
   - For VerificationStore: iterate `fact_store.list_investigations()` and call `verification_store.get_stats(inv_id)` for each investigation, then aggregate totals (total verified, status counts, pending review). VerificationStore does NOT have a global `get_storage_stats()` method — only per-investigation `get_stats(investigation_id)`.
   - Render `monitoring/status.html` with aggregate stats

**api.py** - `router = APIRouter()`:

JSON API endpoints for HTMX partial page updates:

1. `GET /investigation/{investigation_id}/stats` - Return JSON stats for HTMX polling
2. `GET /investigation/{investigation_id}/facts` - Return HTML partial of fact table (for HTMX swap)
