**PREREQUISITE: Add `get_all_classifications` to ClassificationStore.**

Add the following method to `osint_system/data_management/classification_store.py` (modeled on `VerificationStore.get_all_results()`):

```python
async def get_all_classifications(
    self,
    investigation_id: str,
) -> list[dict[str, Any]]:
    """Get all classifications for an investigation.

    Args:
        investigation_id: Investigation identifier.

    Returns:
        List of all classification dicts for the investigation.
    """
    async with self._lock:
        inv = self._storage.get(investigation_id, {})
        return list(inv.get("classifications", {}).values())
```

This is needed by InvestigationExporter._export_classifications and InvestigationArchive.create_archive to enumerate all classification records without relying on get_priority_queue() workarounds. It exposes the full classification dict including all schema fields.

**schema.sql** - SQLite schema definition:

```sql
-- Investigation database schema v1.0
-- Generated by OSINT Multi-Agent System

CREATE TABLE IF NOT EXISTS investigation_metadata (
    investigation_id TEXT PRIMARY KEY,
    objective TEXT DEFAULT '',
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    metadata_json TEXT DEFAULT '{}'
);

CREATE TABLE IF NOT EXISTS facts (
    fact_id TEXT PRIMARY KEY,
    investigation_id TEXT NOT NULL,
    claim_text TEXT NOT NULL,
    assertion_type TEXT DEFAULT 'statement',
    claim_type TEXT DEFAULT 'event',
    content_hash TEXT,
    extraction_confidence REAL,
    claim_clarity REAL,
    temporal_value TEXT,
    temporal_precision TEXT,
    provenance_json TEXT DEFAULT '{}',
    entities_json TEXT DEFAULT '[]',
    stored_at TEXT,
    FOREIGN KEY (investigation_id) REFERENCES investigation_metadata(investigation_id)
);

CREATE TABLE IF NOT EXISTS classifications (
    fact_id TEXT PRIMARY KEY,
    investigation_id TEXT NOT NULL,
    impact_tier TEXT NOT NULL,
    dubious_flags_json TEXT DEFAULT '[]',
    priority_score REAL DEFAULT 0.0,
    credibility_score REAL DEFAULT 0.0,
    classification_reasoning_json TEXT DEFAULT '[]',
    classified_at TEXT,
    FOREIGN KEY (fact_id) REFERENCES facts(fact_id),
    FOREIGN KEY (investigation_id) REFERENCES investigation_metadata(investigation_id)
);

CREATE TABLE IF NOT EXISTS verification_results (
    fact_id TEXT PRIMARY KEY,
    investigation_id TEXT NOT NULL,
    verification_status TEXT NOT NULL,
    original_confidence REAL,
    confidence_boost REAL DEFAULT 0.0,
    final_confidence REAL,
    query_attempts INTEGER DEFAULT 0,
    queries_used_json TEXT DEFAULT '[]',
    supporting_evidence_json TEXT DEFAULT '[]',
    refuting_evidence_json TEXT DEFAULT '[]',
    origin_dubious_flags_json TEXT DEFAULT '[]',
    reasoning TEXT DEFAULT '',
    verified_at TEXT,
    requires_human_review INTEGER DEFAULT 0,
    human_review_completed INTEGER DEFAULT 0,
    FOREIGN KEY (fact_id) REFERENCES facts(fact_id),
    FOREIGN KEY (investigation_id) REFERENCES investigation_metadata(investigation_id)
);

CREATE TABLE IF NOT EXISTS sources (
    source_id TEXT PRIMARY KEY,
    source_domain TEXT,
    source_type TEXT,
    authority_score REAL,
    fact_count INTEGER DEFAULT 0,
    investigation_id TEXT,
    FOREIGN KEY (investigation_id) REFERENCES investigation_metadata(investigation_id)
);

CREATE TABLE IF NOT EXISTS entities (
    entity_id TEXT,
    investigation_id TEXT NOT NULL,
    name TEXT NOT NULL,
    canonical TEXT,
    entity_type TEXT,
    fact_ids_json TEXT DEFAULT '[]',
    PRIMARY KEY (entity_id, investigation_id),
    FOREIGN KEY (investigation_id) REFERENCES investigation_metadata(investigation_id)
);

-- Indexes for common queries
CREATE INDEX IF NOT EXISTS idx_facts_investigation ON facts(investigation_id);
CREATE INDEX IF NOT EXISTS idx_facts_hash ON facts(content_hash);
CREATE INDEX IF NOT EXISTS idx_classifications_tier ON classifications(impact_tier);
CREATE INDEX IF NOT EXISTS idx_classifications_score ON classifications(priority_score DESC);
CREATE INDEX IF NOT EXISTS idx_verification_status ON verification_results(verification_status);
CREATE INDEX IF NOT EXISTS idx_sources_investigation ON sources(investigation_id);
CREATE INDEX IF NOT EXISTS idx_entities_investigation ON entities(investigation_id);
```

**exporter.py** - `InvestigationExporter` class:

Constructor takes:
- `fact_store: FactStore`
- `classification_store: ClassificationStore`
- `verification_store: VerificationStore`
- `output_dir: str = "exports/"`

Uses structlog for logging.

Methods:

1. `async export(investigation_id: str, output_path: str | None = None) -> Path`:
   Main export method. Creates SQLite database at output_path (default: `{output_dir}/{investigation_id}.db`).
   - Read schema.sql from package directory using `importlib.resources` or `Path(__file__).parent / "schema.sql"`
   - Create database with aiosqlite: `async with aiosqlite.connect(db_path) as db:`
   - Execute schema DDL to create all tables
   - Call `_export_metadata`, `_export_facts`, `_export_classifications`, `_export_verification_results`, `_export_sources`, `_export_entities`
   - Log stats (row counts per table)
   - Return Path to created database

2. `async _export_metadata(db: aiosqlite.Connection, investigation_id: str) -> None`:
   Fetch investigation metadata from FactStore. Insert into investigation_metadata table.

3. `async _export_facts(db: aiosqlite.Connection, investigation_id: str) -> int`:
   Fetch all facts via `retrieve_by_investigation`. For each fact:
   - Extract claim.text (or claim["text"]), fact_id, content_hash
   - Serialize provenance and entities as JSON strings
   - Extract extraction_confidence and claim_clarity from quality dict
   - Extract temporal value/precision if present
   - INSERT into facts table
   Return row count.

4. `async _export_classifications(db: aiosqlite.Connection, investigation_id: str) -> int`:
   Fetch all classifications via `classification_store.get_all_classifications(investigation_id)`. For each:
   - Serialize dubious_flags and classification_reasoning as JSON strings
   - INSERT into classifications table
   Return count.

5. `async _export_verification_results(db: aiosqlite.Connection, investigation_id: str) -> int`:
   Fetch all results via `get_all_results`. For each VerificationResultRecord:
   - Use `.model_dump(mode="json")` to get dict
   - Serialize evidence lists and queries as JSON strings
   - INSERT into verification_results table
   Return count.

6. `async _export_sources(db: aiosqlite.Connection, investigation_id: str, facts: list[dict]) -> int`:
   Build source inventory from facts provenance data. Group by source_id, count facts per source. INSERT into sources table. Return count.

7. `async _export_entities(db: aiosqlite.Connection, investigation_id: str, facts: list[dict]) -> int`:
   Extract unique entities from facts' entity lists. Group by canonical name. Track which fact_ids mention each entity. INSERT into entities table. Return count.

**archive.py** - `InvestigationArchive` class:

Constructor takes:
- `fact_store: FactStore`
- `classification_store: ClassificationStore`
- `verification_store: VerificationStore`
- `output_dir: str = "exports/"`

Uses structlog for logging.

Methods:

1. `async create_archive(investigation_id: str, output_path: str | None = None) -> Path`:
   Main method. Creates a JSON archive at output_path (default: `{output_dir}/{investigation_id}_archive.json`).

   Build archive dict:
   ```python
   archive = {
       "schema_version": "1.0",
       "archive_type": "investigation_archive",
       "investigation_id": investigation_id,
       "created_at": datetime.now(timezone.utc).isoformat(),
       "system_version": "0.1.0",
       "data": {
           "investigation_metadata": {...},
           "facts": [...],
           "classifications": [...],
           "verification_results": [...],
       },
       "statistics": {
           "fact_count": N,
           "classification_count": N,
           "verification_count": N,
           "confirmed_count": N,
           "refuted_count": N,
           "dubious_count": N,
       }
   }
   ```

   - Fetch facts from FactStore via `retrieve_by_investigation` -> include raw fact dicts
   - Fetch all classifications via `classification_store.get_all_classifications(investigation_id)` -> list of classification dicts with all schema fields preserved.
   - Fetch verification results via `get_all_results` -> serialize each VerificationResultRecord to dict with `.model_dump(mode="json")`
   - Compute statistics from the collected data
   - Write as indented JSON to output_path
   - Ensure parent directories exist
   - Return Path to created archive

2. `async load_archive(archive_path: str | Path) -> dict[str, Any]`:
   Static/classmethod. Load and validate an archive JSON file:
   - Read JSON file
   - Validate schema_version is supported
   - Return the archive dict
   - Raise ValueError if schema_version unsupported or required keys missing

3. `_compute_statistics(facts: list, classifications: list, verifications: list) -> dict[str, Any]`:
   Count facts, classifications, verifications. Count confirmed/refuted/dubious from verification status values.
