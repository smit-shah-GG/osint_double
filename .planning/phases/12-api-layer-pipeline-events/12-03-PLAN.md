**facts.py** -- `APIRouter(prefix="/api/v1")`:

1. **GET /investigations/{investigation_id}/facts** -- `list_facts(investigation_id, page=1, page_size=100)`:
   - Get fact_store, classification_store, verification_store from app.state.investigation_stores[investigation_id]
   - If investigation_stores does not contain this ID, fall back to checking if stores exist on app.state directly (for serve.py compatibility), else raise NotFoundError
   - Call `fact_store.retrieve_by_investigation(investigation_id)` to get all facts
   - For each fact, enrich with classification and verification data using `_enrich_fact()` helper
   - Return `PaginatedResponse[FactResponse].from_items(enriched_facts, page, page_size)`

2. **GET /investigations/{investigation_id}/facts/{fact_id}** -- `get_fact(investigation_id, fact_id)`:
   - Get fact via `fact_store.get_fact(investigation_id, fact_id)` -- raise NotFoundError if None
   - Enrich with classification and verification
   - Return single `FactResponse`

3. **`_enrich_fact` helper** (per RESEARCH.md "Fact Enrichment" code example):
   - Takes raw fact dict from store + stores + investigation_id
   - Gets classification via `classification_store.get_classification(investigation_id, fact_id)`
   - Gets verification via `verification_store.get_result(investigation_id, fact_id)`
   - Extracts claim text, claim_type from the fact's nested `claim` dict (facts stored as dicts with `claim`, `provenance` sub-dicts)
   - Extracts source_id, source_type from `provenance` dict
   - Returns flat `FactResponse` with all fields populated
   - Handles None gracefully for optional classification/verification (not all facts are classified or verified)

CRITICAL: Internal facts are stored as nested dicts (RESEARCH.md Pitfall 5 -- do NOT expose internal ExtractedFact structure). The `_enrich_fact` function must flatten the nested structure. Look at actual fact_store data format by examining `FactStore.store()` method to understand the dict structure.

1. **GET /investigations/{investigation_id}/reports/latest** -- `get_latest_report(investigation_id)`:

   - Call `report_store.get_latest(investigation_id)` -- raise NotFoundError if None

   - Call `report_store.get_version(investigation_id, version)` -- raise NotFoundError if None
   - Map to ReportResponse

**sources.py** -- `APIRouter(prefix="/api/v1")`:

1. **GET /investigations/{investigation_id}/sources** -- `list_sources(investigation_id, page=1, page_size=100)`:
   - Get article_store from investigation_stores or app.state
   - Call `article_store.retrieve_by_investigation(investigation_id)` to get all articles
   - Aggregate by source domain: for each article, extract `source.name` (domain), `source.type`, `source.authority_score`
   - Group articles by domain, count per domain, take max authority_score per domain
   - Build list of `SourceResponse(name=domain, type=source_type, authority_score=max_score, article_count=count, domain=domain)`
   - Sort by article_count descending (most-used sources first)
   - Return PaginatedResponse

**graph.py** -- `APIRouter(prefix="/api/v1")`:

1. **GET /investigations/{investigation_id}/graph/nodes** -- `list_graph_nodes(investigation_id, node_type: str | None = Query(None))`:
   - Get graph pipeline from `app.state.graph_pipelines.get(investigation_id)` -- raise NotFoundError if not available (graph phase may not have run, or investigation not found)
   - Access the adapter's internal graph: `pipeline._adapter._graph` (NetworkXAdapter stores a networkx.DiGraph)
   - Iterate nodes with `graph.nodes(data=True)` -- each node has attributes including `type`, `label`, etc.
   - Filter by `node_type` if provided
   - Map to `GraphNodeResponse(id=node_id, label=attrs.get("label", node_id), type=attrs.get("type", "unknown"), properties={k:v for k,v in attrs.items() if k not in ("label", "type")})`
   - Return list (no pagination needed -- graphs are small, hundreds of nodes per RESEARCH.md)

2. **GET /investigations/{investigation_id}/graph/edges** -- `list_graph_edges(investigation_id)`:
   - Same graph access pattern
   - Iterate edges with `graph.edges(data=True)`
   - Map to `GraphEdgeResponse(source=u, target=v, relationship=attrs.get("relationship", "related"), properties={...})`
   - Return list

3. **GET /investigations/{investigation_id}/graph/query** -- `query_graph(investigation_id, pattern: str = Query(...), entity_id: str | None = Query(None), from_id: str | None = Query(None), to_id: str | None = Query(None), max_hops: int = Query(2, ge=1, le=5))`:
   - Get graph pipeline adapter
   - Dispatch based on `pattern` parameter:
     - `"entity_network"`: requires entity_id. Call `adapter.query_entity_network(entity_id, max_hops, investigation_id)`
     - `"corroboration"`: Call `adapter.query_corroboration_clusters(investigation_id)`
     - `"timeline"`: requires entity_id. Call `adapter.query_timeline(entity_id, investigation_id)`
     - `"shortest_path"`: requires from_id and to_id. Call `adapter.query_shortest_path(from_id, to_id, investigation_id)`
     - Unknown pattern: raise ProblemDetailError(400, "Invalid query pattern", f"Unknown pattern: {pattern}. Valid: entity_network, corroboration, timeline, shortest_path")
   - QueryResult returned by adapter contains `.nodes` and `.edges` lists -- map these to GraphNodeResponse/GraphEdgeResponse lists
   - Return `{"nodes": [...], "edges": [...]}`
