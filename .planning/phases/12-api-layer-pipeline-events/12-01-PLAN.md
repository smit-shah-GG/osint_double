Create `osint_system/api/events/` package.

**event_models.py** -- typed event structures:

1. `EventType(str, Enum)` with values: `PHASE_STARTED`, `PHASE_PROGRESS`, `PHASE_COMPLETED`, `PIPELINE_COMPLETED`, `PIPELINE_ERROR`. The `.value` should be the SSE event name string (e.g., `"phase_started"`).

2. `PipelineEvent` dataclass:
   - `id: int` (auto-incrementing per investigation)
   - `event_type: str` (the EventType value string, e.g., "phase_started")
   - `data: dict[str, Any]` (event payload -- must be JSON-serializable primitives only)
   - `timestamp: datetime` (default: `datetime.now(timezone.utc)`)

**event_bus.py** -- in-memory pub/sub per RESEARCH.md Pattern 2:

1. `PipelineEventBus` class:
   - `_events: dict[str, list[PipelineEvent]]` -- per-investigation event storage
   - `_subscribers: dict[str, list[asyncio.Queue[PipelineEvent]]]` -- active subscriber queues
   - `_counters: dict[str, int]` -- auto-incrementing event ID per investigation

2. Methods:
   - `emit(investigation_id: str, event_type: str, data: dict) -> PipelineEvent` -- stores event, pushes to all active subscriber queues via `put_nowait`
   - `subscribe(investigation_id: str) -> asyncio.Queue[PipelineEvent]` -- creates and returns new subscriber queue
   - `unsubscribe(investigation_id: str, queue: asyncio.Queue) -> None` -- removes queue from subscriber list
   - `get_events_since(investigation_id: str, last_event_id: int) -> list[PipelineEvent]` -- returns events with id > last_event_id (for reconnection replay)
   - `get_all_events(investigation_id: str) -> list[PipelineEvent]` -- full history
   - `clear(investigation_id: str) -> None` -- remove all events and subscribers for an investigation (cleanup)

3. CRITICAL: All data values in events MUST be JSON-serializable primitives. The emit method should NOT validate this (too expensive), but document it. Event data should use `str`, `int`, `float`, `bool`, `None`, `list`, `dict` only.

**investigation_registry.py** -- lifecycle tracking per RESEARCH.md Pattern 3:

1. `InvestigationStatus(str, Enum)`: `PENDING`, `RUNNING`, `COMPLETED`, `FAILED`, `CANCELLED`

2. `Investigation` dataclass:
   - `id: str`, `objective: str`
   - `status: InvestigationStatus` (default PENDING)
   - `params: dict[str, Any]` (default_factory=dict)
   - `created_at: datetime` (default now UTC)
   - `updated_at: datetime | None = None`
   - `error: str | None = None`
   - `stats: dict[str, int]` (default_factory=dict)

3. `InvestigationRegistry` class:
   - `_investigations: dict[str, Investigation]`
   - `_lock: asyncio.Lock` (for atomic status transitions, per Pitfall 4 in RESEARCH.md)

4. Methods:
   - `create(objective: str, params: dict | None = None, investigation_id: str | None = None) -> Investigation` -- generates UUID-based ID (`inv-{hex[:8]}`), stores and returns
   - `get(investigation_id: str) -> Investigation | None`
   - `list_all() -> list[Investigation]` -- returns all, sorted by created_at descending
   - `async transition(investigation_id: str, expected_status: InvestigationStatus, new_status: InvestigationStatus, error: str | None = None, stats: dict[str, int] | None = None) -> Investigation` -- atomic compare-and-swap with asyncio.Lock. Raises `ConflictError` (from errors.py) if current status != expected_status. Updates `updated_at` timestamp.
   - `delete(investigation_id: str) -> bool` -- removes from registry, returns True if existed

5. Valid transitions: PENDING->RUNNING, RUNNING->COMPLETED, RUNNING->FAILED, RUNNING->CANCELLED, PENDING->CANCELLED. The `transition` method should enforce these (raise ConflictError for invalid transitions like COMPLETED->RUNNING).
