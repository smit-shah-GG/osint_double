**investigations.py** -- `APIRouter(prefix="/api/v1")` with these endpoints:

1. **POST /investigations** (status_code=202) -- `create_investigation(body: LaunchRequest)`:

   - Return 202 with `InvestigationResponse` including `stream_url=f"/api/v1/investigations/{inv.id}/stream"`

2. **GET /investigations** -- `list_investigations(page: int = Query(1, ge=1), page_size: int = Query(100, ge=1, le=1000))`:
   - Get all from registry via `registry.list_all()`
   - Map each Investigation to InvestigationResponse
   - Return `PaginatedResponse.from_items(responses, page, page_size)`

3. **GET /investigations/{investigation_id}** -- `get_investigation(investigation_id: str)`:
   - Get from registry. Raise `NotFoundError` if None
   - Return `InvestigationResponse` with stats from registry

4. **DELETE /investigations/{investigation_id}** -- `delete_investigation(investigation_id: str)`:
   - Check if running -- if so, cancel task first
   - Delete from registry. Raise `NotFoundError` if not found
   - Clean up event bus via `event_bus.clear(investigation_id)`
   - Return 204 No Content

5. **POST /investigations/{investigation_id}/cancel** -- `cancel_investigation(investigation_id: str)`:
   - Get investigation from registry. Raise `NotFoundError` if None
   - Set cancellation flag: `app.state.cancel_flags[investigation_id].set()` (asyncio.Event)
   - Transition status RUNNING -> CANCELLED via registry.transition
   - Emit `pipeline_error` event with `{"reason": "cancelled"}`
   - Return updated InvestigationResponse

6. **POST /investigations/{investigation_id}/regenerate** -- `regenerate_investigation(investigation_id: str, body: RegenerateRequest)`:
   - Get investigation. Raise `NotFoundError` if None
   - Raise `ConflictError` if status is not COMPLETED
   - Launch `_regenerate_pipeline` async task that re-runs only AnalysisPipeline with existing stores
   - Return 202 with updated InvestigationResponse

**_run_pipeline_with_events** -- the event-emitting wrapper around InvestigationRunner (RESEARCH.md anti-pattern: "Do not modify InvestigationRunner directly"):

```python
async def _run_pipeline_with_events(
    investigation_id: str,
    body: LaunchRequest,
    app_state: Any,
) -> None:
```

This function:
1. Creates an `InvestigationRunner(objective=body.objective, investigation_id=investigation_id)` -- the runner creates its own stores internally
2. Creates a cancellation `asyncio.Event` and stores it in `app_state.cancel_flags[investigation_id]`
3. Stores the runner's stores on `app_state` for API read access: `app_state.investigation_stores[investigation_id] = { "fact_store": runner.fact_store, "classification_store": runner.classification_store, ... }`
4. Wraps each pipeline phase call with event emission:
   - Before each phase: `event_bus.emit(investigation_id, "phase_started", {"phase": phase_name})`
   - After each phase: `event_bus.emit(investigation_id, "phase_completed", {"phase": phase_name, "elapsed_ms": ..., **phase_stats})`
   - Progress events between phases with cumulative stats
5. Between each phase, checks `cancel_flags[investigation_id].is_set()` -- if set, skip remaining phases
6. On completion: transition to COMPLETED, emit `pipeline_completed` with final stats
7. On exception: transition to FAILED with error message, emit `pipeline_error`
8. In finally block: remove cancel flag, retain stores for API access
9. Retains `runner.graph_pipeline` (if graph phase ran) in `app_state.graph_pipelines[investigation_id]` for the Graph API endpoint (RESEARCH.md Pitfall 6)

IMPORTANT: The wrapper calls runner phases individually (`runner._phase_crawl()`, `runner._phase_extract()`, etc.) rather than `runner.run()`, because it needs to emit events between phases and check cancellation. This is intentional -- the wrapper IS the orchestrator for API-launched investigations.

**stream.py** -- SSE endpoint using FastAPI built-in SSE (RESEARCH.md: "fastapi.sse.EventSourceResponse, not sse-starlette"):

`APIRouter(prefix="/api/v1")` with one endpoint:

**GET /investigations/{investigation_id}/stream** (response_class=EventSourceResponse):

```python
@router.get("/investigations/{investigation_id}/stream")
async def stream_events(
    request: Request,
    investigation_id: str,
    last_event_id: Annotated[int | None, Header(alias="Last-Event-ID")] = None,
) -> EventSourceResponse:
```

Implementation (per RESEARCH.md Pattern 4):

1. Get investigation from registry. If not found, return EventSourceResponse that yields a single error event and closes.

2. **Replay phase**: Get missed events via `event_bus.get_events_since(investigation_id, last_event_id or 0)`. Yield each as `ServerSentEvent(data=json.dumps(event.data, default=str), event=event.event_type, id=str(event.id))`.

3. **Post-completion check**: If investigation status is terminal (COMPLETED, FAILED, CANCELLED), close the generator after replay. The client gets the full history and knows the pipeline is done. Do NOT subscribe for live events.

4. **Live streaming phase**: Subscribe via `event_bus.subscribe(investigation_id)`. Enter async loop:
   - `await asyncio.wait_for(queue.get(), timeout=30.0)` -- wait for next event
   - On event: yield as ServerSentEvent
   - On terminal event type ("pipeline_completed", "pipeline_error"): break loop
   - On TimeoutError: continue (heartbeat is handled by EventSourceResponse's built-in 15s ping per CONTEXT.md decision)
   - Check `await request.is_disconnected()` periodically to detect client disconnect

5. **Cleanup**: In `finally` block, `event_bus.unsubscribe(investigation_id, queue)`

The endpoint returns `EventSourceResponse(event_generator())` where `event_generator` is the async generator described above.

CRITICAL per RESEARCH.md Pitfall 7: Use `json.dumps(event.data, default=str)` to handle any non-serializable types that slip through.

CRITICAL per RESEARCH.md Pitfall 2: The generator MUST check for terminal event types and break, otherwise the SSE connection stays open forever.
